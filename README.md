🤖 UKnow School Telegram Bot — Архітектура Проекту
Проект побудовано за модульною архітектурою (Domain-Driven Design), що дозволяє легко масштабувати бота, додавати нові функції та підтримувати код у чистоті.

Нижче наведено детальний опис структури директорій та файлів:

📁 1. config/ (Налаштування та константи)
Папка для ізоляції констант, налаштувань та секретів від основної бізнес-логіки.

settings.py: "Мозок" налаштувань. Зчитує токени з .env, зберігає всі глобальні константи, списки (кнопки, мови, тригери) та єдиний реєстр станів (states) для ConversationHandler, щоб уникнути конфліктів імпортів.

__init__.py: Робить папку Python-пакетом для можливості імпорту з неї.

📁 2. database/ (Робота з даними)
Усе, що стосується збереження та отримання інформації з бази даних SQLite.

db_manager.py: Головний клас-менеджер Database. Містить усі SQL-запити: створення таблиць, CRUD-операції для користувачів, уроків, груп та повідомлень.

__init__.py: Дозволяє зручно імпортувати готовий об'єкт db у будь-який інший файл системи.

📁 3. handlers/ (Бізнес-логіка та Роутери)
Найбільший модуль. Тут розділена логіка для різних ролей користувачів та етапів роботи.

admin.py: Панель управління адміністратора. Створення груп, скасування уроків, зміна викладачів, масова розсилка та перегляд статистики.

chat_engine.py: Ядро системи чатів. Керує P2P-спілкуванням (пересилання тексту, медіа, альбомів) між учнями та викладачами/групами, а також генерує історію переписок з пагінацією.

common.py: Спільні обробники. Відповідає за глобальні резервні відповіді (fallback) на невідомий текст, кнопку "Написати менеджеру", спільну навігацію (наприклад, гортання календаря) та кнопку "Назад".

registration.py: Покрокова реєстрація нових користувачів (збір ПІБ, дати народження, мови, телефону) та їх запис до БД.

student.py: Інтерфейс учня. Перегляд власного розкладу, історії листування, правил школи, FAQ та ініціація чату з викладачем.

teacher.py: Інтерфейс викладача. Керування своїм розкладом, перегляд списку учнів і груп. Містить розумні Вхідні (inbox): зручний список учнів з новими повідомленнями та прев'ю тексту (як у месенджерах), можливість переглядати історію та швидко відповідати.

__init__.py: Робить папку Python-пакетом.

📁 4. jobs/ (Фонові задачі)
Система відкладених завдань та роботи за розкладом.

scheduler.py: Налаштування JobQueue. Запускає щоденні автоматичні нагадування учням та викладачам про заплановані уроки (наприклад, о 08:00 ранку).

__init__.py: Робить папку Python-пакетом.

📁 5. services/ (Зовнішні інтеграції)
Міст між ботом та сторонніми сервісами чи файловою системою.

google_sheets.py: Інтеграція з Google Sheets (через Webhook/Google Apps Script) для синхронізації бази даних з таблицями.

backup_service.py: Служба автоматичного або ручного створення резервних копій бази даних (school_bot.db).

__init__.py: Робить папку Python-пакетом.

📁 6. utils/ (Допоміжні інструменти)
Утиліти, які не містять бізнес-логіки, але допомагають іншим модулям.

helpers.py: Помічник. Містить функції для форматування дат, часу, перевірки посилань та інші дрібні логічні фільтри.

keyboards.py: Візуальний інтерфейс бота. Генератори клавіатур (головні меню, інлайн-календарі, кнопки активного чату).

__init__.py: Робить папку Python-пакетом.

📄 Головні файли в корені проекту
main.py: Точка входу в програму (Оркестратор). Налаштовує логування, підключає всі роутери (handlers) до Диспетчера Telegram, ініціалізує фонові задачі та запускає бота.

.env: Локальний файл середовища (не потрапляє в Git). Зберігає найважливіші секрети: BOT_TOKEN, SUPER_ADMIN_ID, URL-и та назву БД.

requirements.txt: Повний перелік зовнішніх бібліотек (залежностей) з їх версіями для розгортання проекту на сервері.

.gitignore: Вказує Git, які файли не потрібно завантажувати в репозиторій (наприклад: .env, *.db, папку backups/, __pycache__).
